HTML

  • Hypertext Markup Language (HTML) gives the content structure
  • Head element contains information (or metadata) about the webpage
  • Body element represents the visible content shown to the user
  • Metadata is used by the browser to serve our webpage and is not typically seen by users
  • Semantic HTML elements describe their content and the role they play on a webpage (e.g. body element is an example of this)
  • Child elements are elements nested inside other elements; whereas a parent element is the element that holds all the children
  • An element refers to the entire element (from opening tag to closing tag); whereas a tag only refers to what appears inside the < brackets

CSS

  • Cascading Style Sheets (CSS) define how the content is presented to users (e.g. colours, aesthetics, visual layout, organisation)
  • There are three ways to style a webpage using CSS (inline, internal and external)
  • External CSS is the most common because it allows developers to keep CSS rules in a separate file
  • When we create an external CSS file, we need to link it to the HTML file using the HTML link element
  • CSS rules are made up of a selector and declaration block
  • We can use HTML elements as selectors (e.g. img) or class attributes (e.g. .card); class attributes allow us to share a CSS rule with more than one element (i.e. we can define a class attribute that applies to multiple elements)
  • Examples of CSS selectors include * (applies rule to all visible elements on the page), img (to any image element), ul (to any list element) and p (to any text)
  • Declarations contain the property (e.g. color) and the value of the property (e.g. blue)
  • Examples of CSS properties include:
  • margin (indicates how much space we want around the outside of an element)
  • padding (indicates how much space we want around the content inside an element)
  • text-align (specifies the horizontal alignment of text in an element; e.g. center)
  • font-size (sets the size of the font)

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add: adds a change in the working branch to the staging area
  • -A: indicates that we want to add all changes
  • git commit -m "message": commits and includes a message that describes what the commit contains (e.g. "add starter code": we're adding our starter code)
  • git pull origin main: checks that are local default branch (i.e. main branch) is in sync with our remote default branch
  • git push origin branch-name: pushes our local changes to our remote branch (i.e. changes made in local branch-name to remote branch-name)
